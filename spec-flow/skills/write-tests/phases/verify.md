# Phase F — gate mechanically, then reconcile (step 9)

Three dispatches: a mechanical-gate leaf, a blind conservation reader, and a cold reconciler. The first repairs what is mechanically wrong; the other two only find — their findings route back through §7, never straight into the diff. Output: `90-verification.md` (the mechanical leaf writes it; the other two append their findings sections).

## The mechanical-gate leaf

Inputs: the suite, the spec_graph, the full frontier chain, **references/rules.md** ("Suite verification"). Reject or repair any test that fails these — they are what makes a test bind, not style. Repair test-level defects in place; anything judgment-shaped is a finding for §7.

- The file **parses**, and every import except the not-yet-written target resolves. The target's import failing is the expected red; do not repair it by committing skeleton source.
- **AST check: every test calls the target symbol** — directly, via a helper in the same file whose body does, or by driving an object a call to the target returned. Repair by inlining the call, never by loosening the check. And no test asserts only against a value its own body computed — the expected side of every assertion traces to the demand or a fixture, never to a re-implementation of the target's logic.
- **Null-stub discrimination.** Run the suite once against a throwaway no-op implementation of the target: **every test must fail, each on its own demand-specific assertion** (an import error or shared fixture crash proves only that the file is broken). A test green against a do-nothing implementation binds nothing — the mechanical kill for vacuous greens, bare negatives included. The stub is never committed. A test that legitimately stays green — a structure, reuse, or parity demand whose target already exists at base — is recorded per-test with its class in `handoff.nullstub_passes`; prefer strengthening it with a self-contained positive control so it fails too. An unexplained pass is vacuous, full stop.
- **Assertions sit at observable seams; fakes inject faults, not policy.** Reject a fake that classifies or decides (any branching beyond returning its spec'd fault) and any assertion that reaches into internals.
- **Every executable demand has its test** — its `discharged_by` resolves to a real test function — **and every gate obligation is discharged** — by a demand or a recorded waiver, **never by a ledger claim** — and every rule has its `gate.evaluated` entry. A probe does not discharge a test: a tolerance that was *probed* and never *pinned* ships untested. If it was probed, it earns a demand.
- **The ledger is closed — and it is IN the artifact.** Every load-bearing claim carries a probe and its observed result, its `probe_kind` matched to its kind; every spend-point cites its claim id; every fake's fault content cites the claim that observed it; a claim left `unprobed` is a finding for §7, not a pass. A claim only the not-yet-written target can settle is marked `deferred` and enumerated in the handoff. A side-file ledger leaves every citation dangling for the cold reviewer, uncheckable.
- **Frontier-chain conservation.** Walk every boundary's inventory against its consumers' `inputs` echoes, end to end: lens premises → synthesis → dispositions → demands → tests, and the brief's flagged facts → premises/claims/`no-consequence` notes. Counts in equal counts out at every edge, every drop named — a consensus answer the suite silently omits is exactly as lost as an obligation no test discharges, and this walk is where it surfaces.
- **No assertion contradicts the ledger.** Scan every test's expected-side literals against the refuted and stale-marked claims: a test asserting content a refutation corrects has pinned the bug green — repair the test to pin the correction, and never resolve the contradiction by weakening the refutation. Neither the null stub (the assertion is true against today's real code) nor conservation (it maps demands to tests, not assertions to claims) can catch this class.
- **Every negative has its paired positive control**, and binds (or explicitly waives) each of its actor's out-edges.
- **The spec_graph checks pass** — `spec-graph binds <artifact>`, `spec-graph actors <artifact> --base <base>`, and `spec-graph claims <artifact>` (`spec-graph` is on your PATH; the plugin ships it). `check_binds` flags a concept threaded in a test docstring (or a clause/waiver's `outcome` prose) but absent from its `binds` — bind the concept so the test must assert it, or record a conscious `binds_waivers:` entry. `check_actors` derives from the CODE — not the design — every CLI/harness/eval entrypoint that drives a changed module, especially a subprocess re-exec that relocates a `PATHS`-style anchor constant onto a different tree, and flags any the graph's `actors` don't model: model the context as an actor, or `actor_waivers:` an out-of-scope one, but never silence a re-exec driver to go green. The input-partition slice (a guard's invalid domain modelled as one bucket) is NOT graph-mechanizable — it wants property-based / mutation testing at impl time (write-code-from-spec §2).
- **Formal slots validate** against schema.md's closed vocabularies: no free text in a query-evaluated slot, no assertion depending on an `nl:` slot. (The closed-vocabulary check stays hand-checked until the spec_graph linter lands; record in `handoff.deviations` that this slice was manual.)

## The blind conservation reader

A reader who has **never seen the intent doc** — dispatch it with the tests alone; the isolation is the instrument, so its prompt must not paraphrase the doc either. Questions are generated from both sides beforehand (the mechanical leaf generates them: one per intent obligation in the obligation's own surface-general terms, one per test — "which obligation needs this?"); the blind reader answers from the tests alone, and the mechanical leaf diffs the answers against the doc. An obligation the tests can't answer is **undischarged intent**; a test no obligation explains is **invented scope or a silently-resolved fork**; "unanswerable" is a first-class verdict, not a reader failure. One open-ended cold read (state the intent the suite implies, then diff) backstops the questions neither side generated.

## The cold reconciler (frontier model, fresh — not the orchestrator, not the phase-E author)

Reads the run's artifact trail — which under this contract is exactly the frontier chain: `10-brief.md` through `70-resolutions.md`, the ledger, the gate record — against the committed diff, and answers one question: *does the spec conserve everything this run learned?* Its charge is the judgment-shaped residue the mechanical checks cannot enumerate: **composition** (two facts probed separately whose intersection no demand binds: a recorder side-effect and a mandated read, each individually pinned, jointly an escape), **altitude** (a demand pinned at a different grain than its premise asked: the builder's empty return pinned where the question was the composed prompt's legibility), and a **`no-consequence` dismissal that looks load-bearing** (conservation confirms the fact was dispositioned; only judgment catches a wrong call). It reads hot on purpose — the mirror of finalize's deliberate cold read: one checks the artifact without the rationale, this one checks the rationale all made it into the artifact.

## Frontier

`90-verification.md` inventory: `{checks_passed: n, repairs: n, findings_for_human: n, nullstub_passes: n}`. Findings route to §7; when the human resolves them, the affected phase (usually E, sometimes D) re-runs against the updated resolutions frontier, and this phase re-verifies. The loop closes when the findings list is empty and every check above is green or recorded.
