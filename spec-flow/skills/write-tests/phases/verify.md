# Phase F — gate mechanically, then reconcile (step 9)

## Topology

- Three dispatches. The first repairs what is mechanically wrong; the other two only find — their findings route through §7, never straight into the diff.
- **Mechanical-gate leaf**: inputs = the suite, the spec_graph, the full frontier chain. Output: `90-verification.md` (it also generates the conservation questions for the blind reader).
- **Blind conservation reader**: inputs = the tests and the generated questions **only** — never the intent doc, and the dispatch prompt must not paraphrase it; the isolation is the instrument. Appends its findings section to `90-verification.md`.
- **Cold reconciler** (frontier model, fresh — never the orchestrator, never the phase-E author): inputs = the full frontier chain, the artifact, the committed diff. Appends its findings section.
- Loop: findings → §7; the human's resolutions re-run the affected phase (usually E, sometimes D); this phase re-verifies. Closed when the findings list is empty and every check is green or recorded.

## Charge — the mechanical-gate leaf

Read **references/rules.md**, "Suite verification". Reject or repair any test that fails these — they are what makes a test bind, not style. Repair test-level defects in place; anything judgment-shaped is a finding for §7.

- The file **parses**, and every import except the not-yet-written target resolves. The target's import failing is the expected red; do not repair it by committing skeleton source.
- **AST check: every test calls the target symbol** — directly, via a helper in the same file whose body does, or by driving an object a call to the target returned. Repair by inlining the call, never by loosening the check. And no test asserts only against a value its own body computed — the expected side of every assertion traces to the demand or a fixture, never to a re-implementation of the target's logic.
- **Null-stub discrimination.** Run the suite once against a throwaway no-op implementation of the target: **every test must fail, each on its own demand-specific assertion** (an import error or shared fixture crash proves only that the file is broken). A test green against a do-nothing implementation binds nothing — the mechanical kill for vacuous greens, bare negatives included. The stub is never committed. A test that legitimately stays green — a structure, reuse, or parity demand whose target already exists at base — is recorded per-test with its class in `handoff.nullstub_passes`; prefer strengthening it with a self-contained positive control so it fails too. An unexplained pass is vacuous, full stop.
- **Assertions sit at observable seams; fakes inject faults, not policy.** Reject a fake that classifies or decides (any branching beyond returning its spec'd fault) and any assertion that reaches into internals.
- **Every executable demand has its test** — its `discharged_by` resolves to a real test function — **and every gate obligation is discharged** — by a demand or a recorded waiver, **never by a ledger claim** — and every rule has its `gate.evaluated` entry. A probe does not discharge a test: a tolerance that was *probed* and never *pinned* ships untested. If it was probed, it earns a demand.
- **The ledger is closed — and it is IN the artifact.** Every load-bearing claim carries a probe and its observed result, its `probe_kind` matched to its kind; every spend-point cites its claim id; every fake's fault content cites the claim that observed it; a claim left `unprobed` is a finding for §7, not a pass. A claim only the not-yet-written target can settle is marked `deferred` and enumerated in the handoff. A side-file ledger leaves every citation dangling for the cold reviewer, uncheckable.
- **Frontier-chain conservation.** Walk every boundary's inventory against its consumers' `inputs` echoes, end to end: lens premises → synthesis → dispositions → demands → tests, and the brief's flagged facts → premises/claims/`no-consequence` notes. Counts in equal counts out at every edge, every drop named — a consensus answer the suite silently omits is exactly as lost as an obligation no test discharges, and this walk is where it surfaces.
- **No assertion contradicts the ledger.** Scan every test's expected-side literals against the refuted and stale-marked claims: a test asserting content a refutation corrects has pinned the bug green — repair the test to pin the correction, and never resolve the contradiction by weakening the refutation. Neither the null stub (the assertion is true against today's real code) nor conservation (it maps demands to tests, not assertions to claims) can catch this class.
- **Every negative has its paired positive control**, and binds (or explicitly waives) each of its actor's out-edges.
- **The spec_graph checks pass** — `spec-graph binds <artifact>`, `spec-graph actors <artifact> --base <base>`, and `spec-graph claims <artifact>` (`spec-graph` is on your PATH; the plugin ships it). `check_binds` runs two scans. **prose ⊄ binds** flags a concept threaded in a test docstring (or a clause/waiver's `outcome` prose) but absent from its `binds` — bind the concept so the test must assert it, or record a conscious `binds_waivers:` entry. **Unexercised seam** flags a demand binding `drives(A->B)` whose test names `B` only inside an `assert`: the demand claims a wiring and the test checks a shape, so it holds whether or not `A` is wired to `B` at all. Drive `A` and assert the observable outcome, or record an `exercise_waivers:` entry. (The class it was forged on: a `parity` demand discharged by `assert isinstance(deps.box, BoxExecutor)`, where the field's own default *is* a `BoxExecutor` — the assertion could not fail, and two bash-enabled roles shipped with no box attached.) `check_actors` derives from the CODE — not the design — every CLI/harness/eval entrypoint that drives a changed module, especially a subprocess re-exec that relocates a `PATHS`-style anchor constant onto a different tree, and flags any the graph's `actors` don't model: model the context as an actor, or `actor_waivers:` an out-of-scope one, but never silence a re-exec driver to go green. The input-partition slice (a guard's invalid domain modelled as one bucket) is NOT graph-mechanizable — it wants property-based / mutation testing at impl time (write-code-from-spec §2).
- **Formal slots validate** against schema.md's closed vocabularies: no free text in a query-evaluated slot, no assertion depending on an `nl:` slot. (The closed-vocabulary check stays hand-checked until the spec_graph linter lands; record in `handoff.deviations` that this slice was manual.)

For the blind reader, generate pointed questions from **both sides** — one per intent obligation, asked in the obligation's own surface-general terms ("can an input vanish with no visible trace?"), and one per test ("which obligation needs this?") — then diff the reader's answers against the doc. An obligation the tests can't answer is **undischarged intent**; a test no obligation explains is **invented scope or a silently-resolved fork**.

Frontier inventory: `{checks_passed: n, repairs: n, findings_for_human: n, nullstub_passes: n}`.

## Charge — the blind conservation reader

You receive a test suite and a list of questions; you have not seen the design and must not seek it — your ignorance is the instrument, and an anchored reader confirms instead of measures. Answer every question from the tests alone. "Unanswerable" is a first-class verdict, not a failure — say it plainly when the tests don't determine an answer. Then one open-ended cold read: state the intent this suite implies, as if reverse-engineering its spec — this backstops the questions neither side generated.

## Charge — the cold reconciler

Read the run's artifact trail — the frontier chain from `10-brief.md` through `70-resolutions.md`, the ledger, the gate record — against the committed diff, and answer one question: *does the spec conserve everything this run learned?* Your charge is the judgment-shaped residue the mechanical checks cannot enumerate: **composition** (two facts probed separately whose intersection no demand binds: a recorder side-effect and a mandated read, each individually pinned, jointly an escape), **altitude** (a demand pinned at a different grain than its premise asked: the builder's empty return pinned where the question was the composed prompt's legibility), and a **`no-consequence` dismissal that looks load-bearing** (conservation confirms the fact was dispositioned; only judgment catches a wrong call). You read hot on purpose — the mirror of finalize's deliberate cold read: one checks the artifact without the rationale, you check the rationale all made it into the artifact.
