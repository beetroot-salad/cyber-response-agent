# Intent+design — port the benign judge's closed-ticket check off bash into a typed host-side tool

Settled in discussion 2026-07-20 (forks resolved: pin-in-body + benign-only registration; bash-side plumbing deleted in this issue). Sequencing: this lands **before** the #665 boxing spec reruns. Cold-reviewed before posting; see §Review.

## Intent — obligations

- **O1** (learning-loop operator): the benign judge keeps its #338 capability — confirm a *cited, closed* past case as policy before a `survived` verdict stands — and the capability survives #665's boxing: the read executes **host-side, in-process**, like the other typed tools.
- **O2** (security — the answer-key defense; negative universal over the surface this change controls): the benign judge **cannot obtain the content of any non-closed ticket through the live ticket-store read** — in particular the open in-flight ticket for the case under judgment. (The pre-existing cached-payload surface is carved out explicitly — see N7.)
- **O3** (security; negative universal): the adversarial judge **cannot reach the ticket store at all**.
- **O4** (operator, surface-general): **no store failure crashes the judge run**: a refused (non-closed/404) ticket, an unreachable store, *or a fault nobody mapped* surfaces to the model as a failed tool result carrying the fault detail — never an unwind out of `agent.iter()`, never a retry loop; the judge's existing "a cited seed the store can't confirm does not survive on that basis" instruction continues to govern.
- **O5** (harness stability): the adapter's CLI surface and its pinned exit codes (0/1/2/64) survive unchanged for the two remaining subprocess consumers (`ticket_seeds._list_closed`, `verify_forward._fetch_closed_resolution`) and the circuit breaker's usage-error class.
- **O6** (unblocks #665): after this lands, the benign judge's bash grant set is exactly `cat` + `defender-sql`; no ticket shape remains on any bash lane.
- **O7** (security — prompt-injection posture): ticket-store content (free-text summaries/resolutions from a remote system) enters the judge **only inside the salted untrusted envelope**, the same posture every adapter payload already rides (`query_tool` wraps both success views and fault detail).

## Non-obligations (examined no's)

- **N1**: no network egress through the box — inherited #665 non-goal; the tool runs on the host, the box stays `--network=none`.
- **N2**: the generic `query` tool is **not** extended to the judge — that would make `require_closed` a model-chosen param with a default (the exact silent-drop the old grant regex existed to prevent) and expose every system's verbs.
- **N3**: no runtime direction check as the adversarial defense — the property is **absence by registration**, not refusal inside the body.
- **N4**: the gather-side surfaces are untouched — the `VERBS` registry and the six CLI-less adapters stay as they are; this adds a fourth consumer over the same two verb bodies, not a rework.
- **N5**: no write path to tickets — the adapter stays read-only (`scripts/case_history/ticket_writer.py` is a separate concern).
- **N6**: extending `defender-policy show` to display typed-tool presence is out of scope — it does not display the `query` bit today either. What this design does guarantee is that presence is a `ToolSet` bit on the built definition, not a conditional buried in a body.
- **N7** (recorded carve-out from O2): investigation-time ticket payloads cached under `gather_raw/` — fetched by the runtime gather, whose `get-ticket` has no closed-only pin — are a **pre-existing surface, identical before and after this change**, and the judge reads `gather_raw/` by design (it is the actuals). O2 governs the live-store read this issue moves; if the cached surface ever needs its own guard, that is a separate issue.
- **Scale dive: does not fire** — a handful of ticket reads per benign judge leg; the transport's mandatory inner timeout replaces the CLI callers' subprocess timeouts, the same tradeoff the `query` tool accepted.

## Design — mechanisms

- **M1 → O1**: a dedicated **deferred typed tool** owned by the judge's package (idiom: `forward_check` in `learning/author/verify_forward/tool.py`), registered through `_register_deferred_tools` (`runtime/tools.py:565`) behind a new `ToolSet` bit (proposal: `closed_tickets`). The body builds a `VerbContext` from `ctx.deps` and calls the **existing verb bodies in-process** off the event loop (idiom: `query` tool, `runtime/query_tool.py:423-446`). **Two model-facing tools**, no operation discriminator: *list closed tickets* (`label`/`q` filters) and *get closed ticket* (`key`).
- **M2 → O2**: the bodies call `list_tickets(..., require_closed=True)` / `get_ticket(..., require_closed=True)` with the flag **hard-coded**; the model-facing schemas carry **no** `require_closed` and **no** `status` parameter. Closed-only moves from *mandatory in the argv grammar* to *unreachable by construction*. The adapter-docstring/spec_graph_611 claim "a params-dict cannot express a mandatory flag" stays true — the param is removed, not made mandatory.
- **M3 → O3**: **benign-only registration**, carried by a **per-leg `tools=` override on the stage-build seam**: `_pydantic_stage.py` already rebinds per-leg `model`/`effort` onto the shared frozen def via `replace(AGENTS[role], model=…, effort=…)` (`learning/pipeline/_pydantic_stage.py:66`), and registration follows `defn.tools` (`runtime/driver.py:218`) — the override rides that same `replace`, set from `JudgeWiring.closed_ticket_read` (`judge/run.py:159-164`). `JUDGE_DEF`'s static default keeps the bit **off**, so the adversarial leg's agent is built without the tool in its schema. (The old carrier, `RunScope.ticket_cli`, reached only `compile_policy`/grants — it never touched registration — and M6 deletes it.)
- **M4 → O4, O7**: the tool bodies mirror the `query` tool's error seam (`runtime/query_tool.py:49-60,316-333`): control-flow exceptions re-raised; `AdapterFault` → its `exit_code`/`detail`; **an unmapped `BaseException` → the fault-class envelope, never an unwind**; every return — success view and fault detail alike — is a plain-string envelope returned as a normal tool result (repo idiom: `_format_bash_result` / `_model_view`) with the payload **salt-wrapped untrusted** (`query_tool.py:403-409`). `ModelRetry` is reserved for malformed calls (missing/ill-formed `key`), mirroring the `query` tool's validator reject.
- **M5 → O5**: the CLI is untouched except its docstring census (three subprocess consumers → two).
- **M6 → O6**: delete the bash-side plumbing and every surface that teaches or checks it:
  - *Mechanism*: `_ticket_grant` + its wiring branch (`engine_pydantic.py:111-112,116-142`); `RunScope.ticket_cli` / `ResolvedRoots.ticket_cli` threading (`agent_definition.py:179-246`); the judge-side carrier, pinned command text, and `_ticket_cli_path` (`judge/run.py:32-57,129,164,186`); `policy_cli.py:52`'s demo scope (which also removes a latent bug — it points at `case_history/case_ticket.py`, not the real CLI).
  - *Teaching surfaces*: rewrite `_cited_policy_read_section` (`judge/run.py:60-94`) **and** the benign system prompt's item 7 (`judge/benign.md:37`) to teach the tool instead of the commands; update `_JUDGE_DENY_REASON` (`engine_pydantic.py:57-68`) — the suite checks it against the live grant list, so the stale "pinned closed-ticket read" clause is test-visible.
  - *Tests*: retire/replace the grant-regex tests (`test_judge_pydantic_engine.py:215-310` spoof defenses, `test_grant_gate_575.py:717-748` e1–e3, `test_bind_sole_seam_551.py:238-262`) and update `learning/test_loop.py:1837-1857` (asserts `scope.ticket_cli is not None` on the benign leg).
  - *Spec graphs*: retire/update the nodes bound to the deleted surface — `spec_graph_575-grant-gate.yaml` (`ticket_grant` e1–e3 at 300-319, taxonomy/witness at 573-580, 729, 894, 966, 1021), `spec_graph_551.yaml:44-45,261`, `spec_graph_538.yaml:366` (RunScope field list), `spec_graph_611-query-tool.yaml:233,362-367,500,540` (the `ticket_cli_dual_surface_survives` nl clause about the judge's bash grant becomes false).
  - *Prose*: comment census — `permission/bash.py:182,331`, `permission/grant.py:196-205` ("Three grants are pins_path exemptions" → two), `policy_cli.py:44-49,77-83`, `_stub_transport.py:78`, `engine_pydantic.py:5-6,102`, `defender/CLAUDE.md` (three exemptions → two).

**Background** (neither obligation nor mechanism): the adapter is deliberately dual-surface — `VERBS` registry + the one surviving CLI (`ticket_adapter.py:2-32`); #611 established the in-process verb-call precedent and killed the other six CLIs; #338 established the benign-policy confirm and why closed-only is its entire security property; the judge is one wiring-parametrized driver with benign/adversarial legs sharing one frozen `AgentDefinition`.

## Security dive (fires)

Assets: the ticket store, and specifically **the open in-flight ticket** (the answer key for the case under judgment). Obligations from the assets: O2, O3, O7. Discharge notes for the spec (prose adversarial review does not discharge these):

- **O2** = safe-by-construction (no model-facing param; probed refusal in the verb bodies — c2/c3) **plus a surface census over live-store paths**: post-M6 the benign judge's model-reachable routes to the store are the two typed tools (closed-only by construction) and nothing else — `cat`/`defender-sql`/`read_file` open mounted files, and the store is HTTP. The census, not the tool body alone, is what the spec should encode. The cached-payload surface is N7, named and out of scope.
- **O3** = path census: no registration on the adversarial build (the `tools=` override fires only when `closed_ticket_read`) + no ticket shape in any bash grant.
- **O7** = the returns ride `untrusted.wrap` — remote free text never enters the judge bare (precedent: every adapter payload and fault detail through the `query` tool).
